I recently had the urge to learn about CPUs and Microprocessors, So I dug down to the roots of how these amazing machines work. We honestly underestimate how much insane engineering goes behind these. But these are not that complex if you understand what exactly goes on.


Lets get the basics down.

From 0s and 1s to performing operations


As we know, the basic language of a cpu is binary i.e. 0 and 1. But how do you perform addition of two numbers just using 0 and 1? We use something called logic gates.


Arrange your everyday switches in a particluar order and you get a logic gate. 0 represents a state of off, and 1 represents a state of on. I won't be diving deeper into those in this blogpost, as it would be too long and boring. Now we arrange these logic gates in a specfic way and then we can add two binary numbers! Combine those together we can add more numbers and do more complex operations.


A small history lesson

This goes all the way back to the 20th century, when Intel came with a breakthrough with their Intel 4004 microprocessor in 1971, marking a significant milestone and laid the foundation for modern CPUs. Shortly afterwards, Intel released a whole family of microprocessors which lead to the personal computer revolution. In 1978, Intel introduced their very first microprocessor, the beloved 8086, which lead the foundation for the x86 architecture which you see in your computers today.

How we actually communicate with these silicon wafers

We communicate with the CPU with instructions They are a combination of commands which tell the CPU what to do. E.g. I want to store my data in the memory of the microprocessor. I will carry this out by giving the cpu an instruction to write a certain memory location with the data, or move it around as we want. Similarly we can also add these numbers and do other operations on them.


Now these instructions are also stored in the memory of the microprocessor, so to execute these we use an instruction cycle. It will fetch the next instruction from memory, decode it and execute it. This cycle repeats till we ask it to stop.


Core Components

Control Unit

Consider this guy as the head of the operations he manages the instruction cycle and coordinates all the tasks performed.

Alithmetic Logic Unit

This guy performs all the required operations given to him by the control unit from the memory like addition, subtraction, and comparisons.

Registers

They are the storage go-downs which are built for easy access of some data which is needed urgently or to store temporarily.


All these together allow the CPU to do some meaningful work.

speed


A CPU has an internal clock which determines how many instructions it can process. This speed is called the /clock speed/. Obviously, the higher the clock-speed the more instructions it can process in a unit of time. In modern CPUs this number is measured in Giga Hertz (GHz), now you know what it exactly means when you have a 5.6 GHz processor, that is 5.6 billion clock cycles in one second! Blazingly fast.


But wait, this is just one guy doing everything, why not more?! Imagine you are doing chores, one guy doing everything is much more time taking than multiple people doing multiple chores at once. Similarly a concept of /cores/ was introduced. The more cores you have, the more tasks you can run at once. For example one core maybe working on the game you are playing while other core is handling the music and so on.
